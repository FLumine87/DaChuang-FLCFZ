# 基于动态跨模态哈希检索和RAG的心理筛查预警系统 - 产品需求文档(PRD)

## 1. 项目概述

### 1.1 项目背景
心理健康问题日益突出，传统心理筛查方式效率低、覆盖面窄、缺乏智能化分析手段。本系统结合**动态跨模态哈希检索**与**RAG（检索增强生成）**技术，通过多模态数据采集（文本、语音、图像），实现高效的心理健康筛查、智能分析与预警。

### 1.2 项目目标
- 构建多模态心理数据采集与管理平台
- 基于跨模态哈希检索实现高效相似案例匹配
- 利用 RAG 技术生成智能分析报告
- 建立分级预警机制，实现心理问题早发现、早干预

### 1.3 目标用户
| 角色 | 说明 |
|------|------|
| 心理咨询师 | 使用系统进行筛查、查看分析报告、管理案例 |
| 管理员 | 管理问卷模板、用户权限、系统配置 |
| 被筛查者 | 填写问卷、上传多模态数据 |

## 2. 功能模块

### 2.1 仪表盘（Dashboard）
| 功能项 | 说明 | 优先级 |
|--------|------|--------|
| 筛查统计总览 | 显示总筛查人数、本月筛查数、预警人数等关键指标 | P0 |
| 预警等级分布 | 饼图/柱状图展示各预警等级人数分布 | P0 |
| 近期筛查趋势 | 折线图展示近30天筛查数量变化趋势 | P1 |
| 最新预警列表 | 展示最近触发的预警记录 | P0 |
| 快捷操作入口 | 新建筛查、查看预警、导出报告等快捷按钮 | P1 |

### 2.2 筛查管理
| 功能项 | 说明 | 优先级 |
|--------|------|--------|
| 问卷模板管理 | 创建、编辑、删除问卷模板（支持量表类型：SCL-90、PHQ-9、GAD-7等） | P0 |
| 筛查任务创建 | 选择问卷模板，设定筛查对象范围，发起筛查任务 | P0 |
| 筛查记录列表 | 查看所有筛查记录，支持按状态、时间、预警等级筛选 | P0 |
| 筛查详情 | 查看单次筛查的详细作答、得分、分析结果 | P0 |

### 2.3 多模态数据采集
| 功能项 | 说明 | 优先级 |
|--------|------|--------|
| 文本采集 | 结构化问卷填写 + 开放式文本输入 | P0 |
| 语音采集 | 录音上传，用于情感分析 | P1 |
| 图像采集 | 绘画测试图片上传（如房树人测试） | P1 |
| 数据预览 | 采集数据的实时预览与确认 | P0 |

### 2.4 跨模态哈希检索与RAG分析
| 功能项 | 说明 | 优先级 |
|--------|------|--------|
| 跨模态检索 | 输入任意模态数据，检索相似历史案例（基于哈希编码的快速匹配） | P0 |
| 检索结果展示 | 展示匹配案例列表，显示相似度、模态来源、关键特征 | P0 |
| RAG智能分析 | 基于检索到的相关案例和知识库，生成综合分析报告 | P0 |
| 分析报告查看 | 结构化展示分析结论、风险评估、建议措施 | P0 |
| 哈希编码可视化 | 可视化展示跨模态哈希编码映射关系（可选） | P2 |

### 2.5 预警管理
| 功能项 | 说明 | 优先级 |
|--------|------|--------|
| 预警规则配置 | 设定各量表/指标的预警阈值和等级（绿/黄/橙/红） | P0 |
| 预警记录列表 | 查看所有预警记录，支持按等级、时间、处理状态筛选 | P0 |
| 预警详情 | 查看触发预警的具体指标、分析依据 | P0 |
| 预警处理流程 | 预警确认、分配处理人、跟进记录、关闭预警 | P1 |
| 预警通知 | 站内消息通知（邮件/短信通知为扩展功能） | P1 |

### 2.6 案例管理
| 功能项 | 说明 | 优先级 |
|--------|------|--------|
| 案例列表 | 查看所有被筛查者档案，支持搜索和筛选 | P0 |
| 案例详情 | 查看个人信息、筛查历史、分析报告、预警记录的综合视图 | P0 |
| 案例时间线 | 以时间线方式展示案例的完整历程 | P1 |
| 案例标签 | 为案例添加标签分类（如焦虑、抑郁、压力等） | P1 |

## 3. 技术架构概要

### 3.1 前端技术栈
- **框架**: React 18 + TypeScript
- **构建工具**: Vite
- **UI框架**: TailwindCSS
- **路由**: React Router v6
- **图表**: Recharts
- **图标**: Lucide React
- **状态管理**: React Context + useReducer

### 3.2 后端技术栈（规划）
- **框架**: Python FastAPI
- **数据库**: PostgreSQL + pgvector
- **向量存储**: FAISS / Milvus
- **哈希检索**: 自研动态跨模态哈希模型
- **RAG引擎**: LangChain + LLM API
- **语音分析**: Whisper + 情感分析模型
- **图像分析**: 预训练视觉模型

## 5. 非功能需求

| 维度 | 要求 |
|------|------|
| 性能 | 页面加载时间 < 2s，哈希检索响应 < 500ms |
| 安全 | 数据加密传输（HTTPS），敏感信息脱敏，权限控制 |
| 可用性 | 界面简洁直观，符合心理健康领域专业规范 |
| 兼容性 | 支持主流浏览器（Chrome、Firefox、Safari、Edge） |
| 扩展性 | 模块化设计，支持后续功能扩展 |

## 6. MVP范围（当前版本）

当前版本聚焦**P0功能**的前端实现，使用Mock数据模拟后端接口：

1. ✅ 仪表盘 - 统计卡片 + 图表 + 预警列表
2. ✅ 筛查管理 - 记录列表 + 详情查看
3. ✅ 多模态数据采集 - 文本输入 + 文件上传界面
4. ✅ 跨模态检索 - 检索界面 + 结果展示 + RAG分析报告
5. ✅ 预警管理 - 预警列表 + 等级标识
6. ✅ 案例管理 - 案例列表 + 案例详情
